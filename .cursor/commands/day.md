# Day Task

Daily execution planning and focus. Structure your day to maximize deep work, handle commitments, and maintain momentum.

## Daily Operating System

Plan your day in three phases: Morning routine ‚Üí Midday check ‚Üí Evening wrap-up

## Morning Routine (30-45 min)

**Best Time:** First thing, before meetings

### STEP 1: Check Critical Signals (10 min)

Check in this order:

```
‚òê Production/System Health
- Status dashboards
- Overnight incidents?
- Customer-reported issues?

‚òê Team Blockers
- Team Slack/async updates
- Anyone blocked?
- Anyone need urgent help?

‚òê Stakeholder Requests
- Leadership emails
- Key stakeholder DMs
- Any "urgent" flags?

‚òê Key Metrics
- Core metrics dashboard
- Unusual changes?
- Compare to yesterday/last week
```

**Decision:** Is anything on fire requiring immediate attention?
- YES ‚Üí Handle the fire, then resume routine
- NO ‚Üí Continue to Step 2

### STEP 2: Review Today's Commitments (10 min)

```
‚òê What meetings do I have today?
- Can any be cancelled/shortened?
- What prep is needed?
- What decisions need to be made?

‚òê What MUST get done today?
- Commitments to others
- Deadlines
- Promised follow-ups

‚òê What can wait if needed?
- Important but not urgent
- Can be delegated
- Can move to tomorrow
```

### STEP 3: Create Today's Focus List (10 min)

Maximum 3-5 items:

```markdown
## Today: [Date]

### MUST DO (Committed)
1. [Task] - [Time estimate] - [Why it matters]
2. [Task] - [Time estimate] - [Why it matters]

### SHOULD DO (High Value)
3. [Task] - [Time estimate]
4. [Task] - [Time estimate]

### COULD DO (If Time)
5. [Task] - [Time estimate]

### DELEGATE/DEFER
- [What I'm moving or giving to others]
```

Use /focus command if struggling to prioritize.

## Evening Wrap-Up (15-20 min)

**Best Time:** Last 20 minutes of work day

### STEP 1: Close the Loop (10 min)

```
‚òê Action items from today's meetings
- Captured in task system
- Scheduled or assigned

‚òê Promises made
- Logged and tracked
- Follow-ups scheduled

‚òê Documents/notes
- Meeting notes shared
- Decisions documented
- Key points sent to stakeholders
```

### STEP 2: Prepare Tomorrow (5 min)

```
‚òê Review tomorrow's calendar
- Any prep needed tonight?
- Any conflicts to resolve?
- Any meetings to move?

‚òê Create tomorrow's tentative focus list
- 3 most important things
- Based on what didn't get done today

‚òê Set yourself up for success
- Documents you'll need
- Questions to think about overnight
- People to talk to
```

### STEP 3: Inbox/Slack Triage (5 min)

```
‚òê Email: Get to inbox zero (or close)
- Respond to urgent
- Defer others with note
- Archive processed items

‚òê Slack: Clear DMs and mentions
- Respond or acknowledge
- Snooze to tomorrow if not urgent
- Clear notification count

‚òê Task list: Updated and prioritized
- Complete items checked off
- Tomorrow's items at top
- Overdue items addressed/rescheduled
```

## Context Switching (Multiple Initiatives)

If managing multiple initiatives, time-block by context:

**Example:**
```
9-11am: [Initiative A] deep work
11-12pm: [BAU] metrics review, team check-ins
1-3pm: [Initiative B] meetings
3-4pm: [BAU] inbox, admin
```

## Common Scenarios

**Scenario: Calendar completely filled with meetings**
- Block focus time FIRST next week
- Cancel/delegate 30% of meetings
- Add "no meeting" blocks to calendar

**Scenario: Fire drill derails morning**
- Handle the fire
- Reassess: What can move to tomorrow?
- Protect afternoon focus block

**Scenario: Can't focus, constant interruptions**
- Set Slack status clearly
- Close email/Slack during focus time
- Communicate boundaries to team

**Scenario: Too many priorities, can't choose**
- Run /focus command
- Use Eisenhower Matrix
- Kill/defer 50%
