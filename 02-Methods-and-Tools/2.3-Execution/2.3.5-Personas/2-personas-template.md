# Persona templates

Use this file as a starting point for each persona. Duplicate it, rename it (e.g. `[persona-name]-persona.md`), and replace all bracketed example text with your own.

## HEART Persona Template

H — Human
- Name:
- Role / Title:
- Experience (years) & Background:

E — Emotions
- Primary Motivation:
- Top Frustration(s):
- How success feels:

A — Actions
- Core daily tasks:
- Typical decisions they own:
- Collaboration style:

R — Results
- Key metrics / goals:
- Constraints (time, tools, rules):
- What’s at stake if they fail:

T — Triggers
- Typical pain point that drives them to act:
- Catalysts that create urgency:
- Main blockers:

Example (brief)
- Name: [Persona Name] — [Role]
- Motivation: [What they care about most in their work]
- Daily: [3–5 key activities or workflows]
- Key metric: [How they measure success]

# JOBS Persona Template

Job to be Done
- Core job (single sentence; start with "When X, I want to Y, so I can Z"):

Trigger
- What event or situation prompts this job?

Outcome (Success measures)
- How does the user measure success? (metrics, feelings, time saved)

Obstacles
- Main barriers, technical or organizational

Context
- Frequency, environment, dependencies, decision rules

Example (brief)
- Job: When [trigger/situation], I want to [job/goal] so I can [desired outcome].
- Trigger: [Key event or situation that makes this job relevant].
- Outcome (Success measures): [What "good" looks like, in their words and/or metrics].
