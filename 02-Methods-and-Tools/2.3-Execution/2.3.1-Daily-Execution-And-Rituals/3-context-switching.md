# Context Switching Guide

## Overview

This guide helps PMs manage multiple initiatives, stakeholders, and priorities by creating clear context zones and minimizing the cognitive cost of switching between different areas of work.

## Context Switching Guide

### Managing Multiple Initiatives

**Reality:** PMs rarely work on just one thing. You're juggling multiple initiatives, stakeholders, and priorities.

-----

### STEP 1: Map Your Context Zones

**Identify your distinct areas of work:**

```markdown
## My Context Zones

### ZONE 1: [Project/Initiative Name]
- **Time allocation:** [X%]
- **Status:** [Phase of work]
- **Key people:** [Stakeholders]
- **Next milestone:** [Date and deliverable]

### ZONE 2: [Project/Initiative Name]
- [Same structure]

### ZONE 3: [BAU/Operations]
- **Time allocation:** [X%]
- **Includes:** Metrics reviews, team management, customer feedback

### ZONE 4: [Strategic/Future]
- **Time allocation:** [X%]
- **Includes:** Discovery, planning, stakeholder relationships
```

**Ideal split:** 70% execution, 20% operations, 10% future/strategy

-----

### STEP 2: Time Block by Context

**Don't mix contexts in same time block**

**Example Weekly Schedule:**

```
MONDAY
- 9-11am: ZONE 1 (Project A deep work)
- 11-12pm: ZONE 3 (Metrics review, team 1-on-1s)
- 1-3pm: ZONE 2 (Project B meetings)
- 3-4pm: ZONE 3 (Inbox/Slack, admin)

TUESDAY
- 9-11am: ZONE 2 (Project B deep work)
- 11-12pm: ZONE 4 (Discovery interviews)
- 1-2pm: ZONE 1 (Project A stakeholder sync)
- 2-4pm: ZONE 1 (Project A work)

[Continue for week]
```

**Benefits:**

- Reduces context switching cost
- Allows deeper focus
- Makes saying no easier ("That's not my focus today")

-----

### STEP 3: Create Context Switching Rituals

**Between contexts, take 5-10 minutes:**

**CLOSE previous context:**

```
‚òê Capture open loops (write down where you left off)
‚òê Document next steps
‚òê Close related tabs/documents
‚òê Clear mental state
```

**OPEN next context:**

```
‚òê Review where you left off
‚òê Check priority for this context
‚òê Open relevant documents
‚òê Set intention (what you want to accomplish)
```

**Example ritual:**

```
Closing Project A:
- Save current work
- Write "Next: Review design mocks" in task
- Close Figma, Slack channels
- Take 2-minute break

Opening Project B:
- Read yesterday's notes
- Check "Today's focus" for Project B
- Open PRD and Jira
- Set timer for 90-min focus block
```

### STEP 4: Triage Interruptions

**When something urgent comes up:**

**Interrupt Decision Tree:**

```
Is this actually urgent?
‚îî‚îÄ NO ‚Üí Add to appropriate context zone, handle later
‚îî‚îÄ YES ‚Üí Ask: Can it wait until my next context switch?
‚îî‚îÄ YES ‚Üí Schedule for next appropriate time block
‚îî‚îÄ NO ‚Üí Handle now, but:
1. Close current context properly
2. Handle interruption
3. Decide: Return to previous context or move on?
```

**Response templates:**

**"Not urgent":**

> "Adding this to my [Project X] time tomorrow. Will update you by EOD tomorrow."

**"Can wait 2 hours":**

> "I'm in focused work until 2pm. Can we discuss at 2:15? If it can't wait, let me know and I'll switch contexts."

**"Truly urgent":**

> "On it. Give me 5 min to wrap what I'm doing, then I'm fully on this."
