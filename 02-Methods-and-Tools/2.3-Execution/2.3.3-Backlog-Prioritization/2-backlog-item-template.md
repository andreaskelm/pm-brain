# Backlog Item Template

Use this template to document backlog items with scoring and rationale. Copy this template for each backlog item you're prioritizing.

```markdown
## [Item Title]

**Type:** [Initiative | Feature | Improvement | Tech Debt | Bug]
**Status:** [Backlog | Ready | In Progress | Done]
**Owner:** [Name]
**Requestor:** [Name/Team]

### Description
[1-2 sentence description of what this is]

### Why (Strategic Alignment)
- **Supports:** [Which OKR/goal?]
- **User need:** [What problem does it solve?]
- **Evidence:** [Links to research, data, feedback]

### Scoring
- **Method:** [RICE | ICE | Value/Effort | SeverityÃ—Frequency]
- **Scores:**
  - [Dimension 1]: [Score] - [Rationale]
  - [Dimension 2]: [Score] - [Rationale]
  - [Dimension 3]: [Score] - [Rationale]
- **Priority Score:** [Calculated score]

### Effort Estimate
- **Size:** [S | M | L | XL]
- **Rough estimate:** [Days/weeks]
- **Dependencies:** [What needs to happen first?]

### Trade-offs
**If we do this, we won't do:** [What gets deprioritized?]

### Links
- PRD: [Link if exists]
- Research: [Link to user research]
- Data: [Link to supporting data]
- Design: [Link to mockups]
```

## How to Use

1. Copy this template for each backlog item
2. Fill in all sections with specific information
3. Use appropriate scoring method based on item type (see framework guide)
4. Document rationale for scores to enable defensible prioritization
5. Link to supporting evidence (research, data, PRDs)
